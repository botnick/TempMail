Authentication — SDK & Internal

ทุก endpoint ใน /v1/* และ /internal/* ต้องส่ง API Key:

Authorization: Bearer <API_KEY>

Source: api/main.go → apiTokenMiddleware()

Validation Flow

1. Hash API key ด้วย SHA-256
2. ตรวจสอบ hash กับ Redis set system:api_key_hashes
3. ถ้า key มี flag isInternal: true → อนุญาตเข้า /internal/* routes

Error ถ้าไม่มี/ผิด

// 401 Unauthorized
{ "error": "Missing or invalid API key" }

Authentication — Admin

ทุก endpoint ใน /admin/* (ยกเว้น /admin/login) ต้องมี session cookie:

Cookie: admin_session=<HMAC_SIGNED_TOKEN>

Source: api/main.go → adminSessionMiddleware()

// 401 Unauthorized

Internal Endpoint — Mail Ingest

POST /internal/mail/ingest — รับ email จาก mail-edge

⚠ ต้องใช้ API Key ที่มี isInternal: true

Request Body

{
  "from": "sender@example.com",
  "to": ["recipient@yourdomain.com"],
  "rawEmail": "<base64-encoded RFC822 email>"
}

Response 200 OK

{
  "status": "accepted",
  "mailbox": "test@example.com",
  "messageId": "uuid"
}

Error Responses

StatusCodeMessage
400invalid_requestInvalid request body
400no_recipientsNo recipients
400invalid_emailFailed to decode raw email
404mailbox_not_foundNo active mailbox found
422spam_rejectedMessage rejected by spam filter
500storage_errorFailed to upload to storage
500database_errorDatabase error

Side Effects

• Upload raw email → R2
• Upload attachments → R2
• Sanitize HTML (ลบ script, iframe, event handlers)
• ส่ง SSE event mailbox:{id}
• Fire webhook (ถ้าตั้งค่าไว้)

Node.js SDK Client

// tempmail-client.js — ใช้ฝั่ง server เท่านั้น!
const API_URL = process.env.TEMPMAIL_API_URL;
const API_KEY = process.env.TEMPMAIL_API_KEY;

async function tmFetch(path, method = 'GET', body = null) {
  const opts = {
    method,
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
      'Content-Type': 'application/json'
    },
  };
  if (body) opts.body = JSON.stringify(body);
  const res = await fetch(`${API_URL}${path}`, opts);
  if (!res.ok) {
    const err = await res.json().catch(() => ({}));
    throw new Error(err.error?.message || `API error: ${res.status}`);
  }
  return res.json();
}

export const getDomains = () => tmFetch('/v1/domains');
export const createMailbox = (d, ttl) => tmFetch('/v1/mailbox/create','POST',{domain:d,ttl});
export const getMailbox = (id) => tmFetch(`/v1/mailbox/${id}`);
export const getMessages = (id, limit=50) => tmFetch(`/v1/mailbox/${id}/messages?limit=${limit}`);
export const getMessage = (id) => tmFetch(`/v1/message/${id}`);
export const deleteMailbox = (id) => tmFetch(`/v1/mailbox/${id}`, 'DELETE');
export const deleteMessage = (id) => tmFetch(`/v1/message/${id}`, 'DELETE');
export const extendMailbox = (id, ttl) => tmFetch(`/v1/mailbox/${id}`, 'PATCH', {ttl});

Security Checklist

✅รายการ
✅เรียก API จาก server-side เท่านั้น — ห้ามเอา API_KEY ไปใส่ browser
✅เก็บ mailboxId ใน DB ผูกกับ userId — ตรวจสิทธิ์ก่อนเข้าถึง
✅UUID = เดา ID ไม่ได้, ไม่เรียงลำดับ
✅HTML ถูก sanitize แล้ว (bluemonday) — XSS-safe
✅ตั้ง TTL ให้สั้น — ลดความเสี่ยงข้อมูลรั่ว
✅API Key hash ด้วย SHA-256 — ไม่เก็บ plaintext ใน DB
✅Admin session signed ด้วย HMAC-SHA256
✅Webhook payload signed ด้วย HMAC-SHA256

📧 TempMail — คู่มือ API

เอกสาร API ฉบับภาษาไทย เขียนจาก source code จริง · ครบทุก endpoint + status code

Base URL: http://your-server:3000

🔐 การยืนยันตัวตน (Authentication)

สำหรับ SDK / Internal (/v1/*, /internal/*)

ส่ง API Key ผ่านทาง Header อย่างใดอย่างหนึ่ง:

วิธีHeaderตัวอย่าง
X-API-KeyX-API-KeyX-API-Key: your-api-key
Bearer TokenAuthorizationAuthorization: Bearer your-api-key

สำหรับ Admin (/admin/*)

1. เรียก POST /admin/login ก่อน (ไม่ต้องใส่ auth)

2. ใช้ token ที่ได้จาก login เป็น Bearer Token สำหรับทุก route อื่น:

Authorization: Bearer <session-token>
⏰ Session token หมดอายุภายใน 24 ชั่วโมง

❌ รูปแบบ Error Response

แบบมาตรฐาน (SDK routes ส่วนใหญ่)

{
  "error": "Human readable error message"
}

แบบมี Code (Admin routes + Ingest)

{
  "error": {
    "code": "error_code",
    "message": "Human readable error message"
  }
}

⏱️ Rate Limiting (จำกัดจำนวนคำขอ)

ขอบเขตค่าเริ่มต้นช่วงเวลา
Public routes (/health)60 ครั้ง1 นาที / IP
Login (/admin/login)5 ครั้ง1 นาที / IP

เมื่อเกินจำนวน → ได้รับ 429:

{
  "error": "Rate limit exceeded. Try again later."
}

🔒 Authentication Errors ที่ใช้กับทุก endpoint

StatusสาเหตุResponse
401ไม่ได้ส่ง API key (SDK)"API key required"
401API key ไม่ถูกต้อง (SDK)"Invalid API key"
401Session หมดอายุ / ไม่ถูกต้อง (Admin)"Invalid or expired session"
500ยังไม่ได้ตั้งค่า ADMIN_API_KEY"Admin panel not configured"
429เรียกถี่เกินไป"Rate limit exceeded. Try again later."

GET/api/server-info

ดึงข้อมูลเซิร์ฟเวอร์และ DNS Instructions

ดึงข้อมูล server node + DNS records ที่ต้องตั้งค่าเมื่อเพิ่ม domain ใหม่ — ไม่ต้องยืนยันตัวตน

💡 Web App ใช้ endpoint นี้ดึงข้อมูลไปแสดง DNS setup instructions ให้ user

ตัวอย่าง Request

GET /api/server-info

Response สำเร็จ — 200

{
  "hostname": "mx1.tempmail.dev",
  "ip": "68.183.184.209",
  "smtp_port": 25,
  "dns_records": {
    "mx": [
      { "type": "MX", "name": "@", "value": "mx1.tempmail.dev", "priority": 10 }
    ],
    "spf": {
      "type": "TXT",
      "name": "@",
      "value": "v=spf1 ip4:68.183.184.209 ~all"
    },
    "dmarc": {
      "type": "TXT",
      "name": "_dmarc",
      "value": "v=DMARC1; p=none;"
    }
  },
  "nodes": [
    {
      "id": "uuid",
      "name": "primary",
      "hostname": "mx1.tempmail.dev",
      "ip": "68.183.184.209",
      "region": "sgp1",
      "active": true
    }
  ]
}

คำอธิบายฟิลด์

ฟิลด์ชนิดคำอธิบาย
hostnamestringHostname ของ node หลัก (ถ้าไม่มี Hostname จะใช้ IP แทน)
ipstringIP address ของ node หลัก
smtp_portintegerSMTP port — ตั้งค่าผ่าน setting smtp_port (ค่าเริ่มต้น: 25)
dns_records.mxarrayMX records — 1 record ต่อ active node, priority เพิ่มทีละ 10
dns_records.spfobjectSPF TXT record — รวม IP ทุก node, qualifier ตั้งผ่าน spf_qualifier
dns_records.dmarcobjectDMARC TXT record — policy ตั้งผ่าน dmarc_policy
nodesarrayรายการ server nodes ที่ active ทั้งหมด

⚙️ ตั้งค่าได้ผ่าน Admin Panel → Settings

Keyค่าเริ่มต้นคำอธิบาย
smtp_port25SMTP port ที่แสดงใน DNS instructions
dmarc_policynoneDMARC policy: none, quarantine, reject
spf_qualifier~allSPF qualifier: ~all, -all, +all, ?all

ตัวอย่างการใช้งานใน JavaScript

const res = await fetch('http://68.183.184.209:4000/api/server-info');
const info = await res.json();

// แสดง DNS instructions ให้ user
console.log(`MX Record: ${info.dns_records.mx[0].value}`);
console.log(`SPF Record: ${info.dns_records.spf.value}`);
console.log(`DMARC Record: ${info.dns_records.dmarc.value}`);

POST/v1/mailbox/create

สร้างกล่องจดหมายชั่วคราว

สร้าง mailbox ใหม่ สามารถกำหนดชื่อเองหรือปล่อยให้ระบบสร้างอัตโนมัติ

Request Body

{
  "localPart": "john.doe",
  "domainId": "uuid-of-domain",
  "tenantId": "user-123",
  "ttlHours": 24
}
ฟิลด์ชนิดจำเป็นคำอธิบาย
localPartstringไม่ชื่อ mailbox (a-z, 0-9, . - _) — ถ้าไม่ใส่จะสร้างอัตโนมัติ
domainIdstringไม่UUID ของโดเมนที่ต้องการ — ถ้าไม่ใส่จะใช้โดเมน public ตัวแรก
tenantIdstringไม่ID ผู้ใช้ของคุณ — ค่าเริ่มต้น: "anonymous"
ttlHoursintegerไม่อายุการใช้งาน (ชั่วโมง) — ค่าเริ่มต้น: 24

Response สำเร็จ — 201

{
  "id": "uuid",
  "address": "john.doe@example.com",
  "localPart": "john.doe",
  "domain": "example.com",
  "domainId": "uuid",
  "expiresAt": "2026-03-15T01:53:51Z",
  "status": "ACTIVE"
}

POST/internal/ingest

รับอีเมลเข้าสู่ระบบ (Mail Ingestion)

mail-edge ส่ง parsed email เข้ามาเก็บในระบบ — รองรับ attachments, spam scoring, และ webhook notification

Request Body

{
  "to": "john.doe@example.com",
  "from": "sender@gmail.com",
  "subject": "Hello World",
  "textBody": "Plain text...",
  "htmlBody": "<p>HTML content</p>",
  "rawEmail": "base64-encoded-raw-email",
  "spamScore": 0.5,
  "spamAction": "no action",
  "attachments": [
    {
      "filename": "doc.pdf",
      "contentType": "application/pdf",
      "content": "base64-encoded-content"
    }
  ]
}

Response สำเร็จ — 201

{
  "status": "stored",
  "messageId": "uuid",
  "mailboxId": "uuid"
}

Status Codes ทั้งหมด

StatusCodeความหมาย
201—เก็บอีเมลสำเร็จ
400invalid_requestMissing required fields: to, from
400invalid_recipientInvalid recipient format
400message_too_largeอีเมลใหญ่เกินกำหนด (ค่าเริ่มต้น 25 MB)
400too_many_attachmentsไฟล์แนบเกินจำนวนที่กำหนด
400attachment_too_largeไฟล์แนบตัวเดียวใหญ่เกิน

404mailbox_not_foundไม่พบ mailbox ผู้รับ (or inactive/expired)
403domain_blockedโดเมนผู้ส่งถูกบล็อก
422spam_rejectedคะแนน spam เกิน threshold

🔔 Webhook Payloads

เมื่อตั้งค่า webhook_url ใน settings ระบบจะส่ง POST request เมื่อมีข้อความใหม่เข้า:

Headers ที่ส่งไป

Content-Type: application/json
X-Webhook-Event: message.received
X-Webhook-Signature: sha256=<hmac-hex>  // ถ้าตั้ง webhook_secret ไว้

Payload

{
  "event": "message.received",
  "mailboxId": "uuid",
  "messageId": "uuid",
  "to": "john@example.com",
  "from": "sender@gmail.com",
  "subject": "Hello World",
  "timestamp": "2026-03-14T01:53:51Z"
}
💡 ตรวจสอบ signature ด้วย HMAC-SHA256 ของ payload body กับ webhook_secret

Security Model

✅หลักการ
✅เรียก API จาก server-side เท่านั้น — ห้ามเอา API_KEY ไปใส่ใน browser JavaScript
✅เก็บ mailboxId ใน DB ของคุณ ผูกกับ userId — ตรวจสิทธิ์ทุกครั้ง
✅Mailbox ID เป็น UUID — เดาไม่ได้ ไม่เรียงลำดับ
✅HTML content ถูก sanitize แล้ว (XSS-safe) ก่อนเก็บ
✅ใช้ tenantId (userId ของคุณ) ตอนสร้าง mailbox — สำหรับ audit trail
✅ตั้ง ttlHours ให้สั้น — ลดความเสี่ยงข้อมูลรั่ว

สถาปัตยกรรม — User-Triggered Fetch (ไม่ใช่ Polling)

⛔ ห้ามใช้ Polling! การ poll ทุก N วินาทีสิ้นเปลือง resource ทั้งสองฝั่ง

✅ แนะนำ: User กดเอง

1. User เปิดหน้า inbox → เว็บเรียก GET /v1/mailbox/{id}/messages
2. User กด "Refresh"   → เรียกใหม่
3. User กดอ่านเมล์     → เรียก GET /v1/message/{msg_id}

✅ ทางเลือก: Auto-refresh แบบ Backoff (ถ้าจำเป็น)

// Poll เฉพาะตอน user รอเมล์อยู่ในหน้า inbox
async function waitForEmail(mailboxId, maxWaitMs = 120000) {
  const start = Date.now();
  let interval = 3000; // เริ่มที่ 3 วินาที

  while (Date.now() - start < maxWaitMs) {
    const { messages } = await fetchMessages(mailboxId);
    if (messages.length > 0) return messages;

    await sleep(interval);
    interval = Math.min(interval * 1.5, 10000); // สูงสุด 10 วิ
  }
  return []; // หมดเวลา
}
// หยุด poll ทันทีเมื่อ user ออกจากหน้า
กฎ: Poll เฉพาะตอน user อยู่หน้า inbox | หยุดทันทีเมื่อออก | Backoff: 3s → 4.5s → 6.75s → 10s | Timeout สูงสุด 2 นาที

Node.js Client (Server-Side)

// tempmail-client.js — ใช้ฝั่ง server เท่านั้น ห้ามเอา API_KEY ไปใส่ browser!

const API_URL = process.env.TEMPMAIL_API_URL;
const API_KEY = process.env.TEMPMAIL_API_KEY;

async function tmFetch(path, method = 'GET', body = null) {
  const opts = {
    method,
    headers: { 'X-API-Key': API_KEY, 'Content-Type': 'application/json' },
  };
  if (body) opts.body = JSON.stringify(body);

  const res = await fetch(`${API_URL}${path}`, opts);
  if (!res.ok) {
    const err = await res.json().catch(() => ({}));
    throw new Error(err.error || `TempMail API error: ${res.status}`);
  }
  return res.json();
}

// ─── Exported Functions ──────────────────────────

export async function getDomains() {
  return tmFetch('/v1/domains');
}

export async function createMailbox(userId, localPart = '', domainId = '', ttlHours = 24) {
  return tmFetch('/v1/mailbox/create', 'POST', {
    localPart, domainId, tenantId: userId, ttlHours
  });
}

export async function getMessages(mailboxId) {
  return tmFetch(`/v1/mailbox/${mailboxId}/messages`);
}

export async function readMessage(messageId) {
  return tmFetch(`/v1/message/${messageId}`);
}

export async function deleteMailbox(mailboxId) {
  return tmFetch(`/v1/mailbox/${mailboxId}`, 'DELETE');
}

export async function getMailbox(mailboxId) {
  return tmFetch(`/v1/mailbox/${mailboxId}`);
}

export async function extendMailbox(mailboxId, ttlHours = 24) {
  return tmFetch(`/v1/mailbox/${mailboxId}`, 'PATCH', { ttlHours });
}

export async function deleteMessage(messageId) {
  return tmFetch(`/v1/message/${messageId}`, 'DELETE');
}

export async function getAttachment(attachmentId) {
  return tmFetch(`/v1/attachment/${attachmentId}`);
}

export async function getMailboxCount(tenantId) {
  return tmFetch(`/v1/mailbox/count?tenantId=${tenantId}`);
}

ตัวอย่างใช้ใน Route Handler

import { createMailbox, getMessages, readMessage } from './tempmail-client.js';

// POST /api/inbox/create
app.post('/api/inbox/create', async (req, res) => {
  const userId = req.user.id; // จาก auth ของคุณ
  const { name, domain } = req.body;

  const mailbox = await createMailbox(userId, name, domain, 48);

  // เก็บ mailbox.id ใน DB ของคุณ ผูกกับ userId
  await db.userMailboxes.create({
    userId,
    mailboxId: mailbox.id,
    address: mailbox.address,
    expiresAt: mailbox.expiresAt,
  });

  res.json({ address: mailbox.address, expiresAt: mailbox.expiresAt });
});

// GET /api/inbox/:mailboxId/messages
app.get('/api/inbox/:mailboxId/messages', async (req, res) => {
  // ⚠ สำคัญ: ตรวจว่า mailbox นี้เป็นของ user ที่ login อยู่
  const ownership = await db.userMailboxes.findOne({
    where: { mailboxId: req.params.mailboxId, userId: req.user.id }
  });
  if (!ownership) return res.status(403).json({ error: 'Not your mailbox' });

  const data = await getMessages(req.params.mailboxId);
  res.json(data);
});
⚠ ต้องตรวจสิทธิ์ mailbox ทุกครั้ง ก่อนดึงข้อมูล — ไม่งั้น user คนอื่นอาจเข้าถึง mailbox ที่ไม่ใช่ของตัวเอง

Security Checklist

✅รายการ
✅เรียก API จาก server-side เท่านั้น — ห้ามเอา API_KEY ไปใส่ browser
✅เก็บ mailboxId ใน DB ผูกกับ userId — ตรวจสิทธิ์ก่อนเข้าถึงทุกครั้ง
✅ห้าม user ส่ง mailboxId อิสระโดยไม่ตรวจ ownership
✅ใช้ tenantId (userId) ตอนสร้าง mailbox — audit trail
✅แสดงเตือน spam (isSpam: true) แต่ไม่ต้องซ่อนเมล์
✅ตั้ง ttlHours ให้สั้นพอ — ลดช่วงเวลาที่ข้อมูลถูกเข้าถึงได้
✅ไม่ poll — ใช้ user-triggered fetch หรือ backoff pattern

Error Handling

ทุก error คืน JSON format เดียวกัน:

{ "error": "Human-readable error message" }
HTTP Statusความหมายวิธีแก้
400Request ผิด (parameter ไม่ถูก)ตรวจ request body/params
401ไม่มี X-API-Key หรือ key ผิดตรวจ API key
404ไม่พบ mailbox / messagemailbox อาจหมดอายุแล้ว
409ชื่อซ้ำ (Conflict)ลองใช้ชื่ออื่น
429Rate limitedรอสักครู่แล้วลองใหม่
500Server errorลองใหม่ + ตรวจ log TempMail

Limits ของระบบ

Limitค่าเริ่มต้นคำอธิบาย
ขนาดเมล์สูงสุด25 MBเมล์ใหญ่กว่านี้โดน reject ที่ SMTP
ไฟล์แนบสูงสุด10 ไฟล์ไฟล์เกินจะถูกตัดออก
ขนาดไฟล์แนบ10 MBต่อไฟล์
Spam rejectคะแนน > 15โดน reject ตั้งแต่ SMTP ไม่เข้าระบบ
เมล์ต่อ mailbox100 ฉบับคืนล่าสุด 100 ฉบับ
ความยาวชื่อ1-64 ตัวอนุญาต: a-z 0-9 . - _

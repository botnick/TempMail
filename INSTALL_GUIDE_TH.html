TempMail — คู่มือติดตั้งและใช้งาน

คู่มือฉบับละเอียด อ่านตามทีละขั้น ใช้งานได้จริง

ระบบนี้คืออะไร?

TempMail เป็น backend service สำหรับรับอีเมลชั่วคราว เว็บหลักของคุณแค่เรียก API 2 บรรทัด:

API_URL = https://api.example.com
API_KEY = YOUR_KEY

ระบบจะรับเมล์จากอินเทอร์เน็ต → กรอง spam → เก็บข้อมูล → ส่งให้เว็บหลักผ่าน API → ลบอัตโนมัติเมื่อหมดอายุ

อินเทอร์เน็ต ── port 25 ──▶ mail-edge (รับเมล์) │ Rspamd (กรอง spam) │ api:4000 (เก็บข้อมูล) │ ┌──────────────┼──────────────┐ Postgres Redis R2 (ฐานข้อมูล) (cache/queue) (เก็บไฟล์) │ worker (ลบเมล์หมดอายุ)

ขั้นตอนที่ 1 — เตรียมเซิร์ฟเวอร์

SSH เข้าเซิร์ฟเวอร์ แล้วรันคำสั่งตามนี้ทีละบรรทัด:

1
อัปเดต OS ให้เป็นเวอร์ชันล่าสุด
sudo apt update && sudo apt upgrade -y
คำสั่งนี้จะดาวน์โหลดและติดตั้งอัปเดตความปลอดภัยทั้งหมด ใช้เวลาประมาณ 2-5 นาที
2
ตั้ง timezone เป็น UTC
sudo timedatectl set-timezone UTC
ตั้ง timezone เป็น UTC เพื่อให้เวลาใน log ตรงกันทุก node ป้องกันปัญหาเมล์หมดอายุผิดเวลา
3
รีบูต
sudo reboot
รีบูตเพื่อใช้ kernel ใหม่ รอสัก 30 วินาที แล้ว SSH กลับเข้ามาใหม่
4
สร้าง Swap (ถ้า RAM 1 GB)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
Swap คือพื้นที่ดิสก์ที่ใช้แทน RAM เมื่อ RAM เต็ม ถ้าเครื่องมี RAM 2 GB ขึ้นไป ข้ามขั้นตอนนี้ได้
5
ติดตั้ง Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
Docker ใช้รัน containers ทั้งหมดของระบบ คำสั่ง usermod อนุญาตให้ user ปัจจุบันใช้ Docker ได้โดยไม่ต้อง sudo
6
ออกจากระบบแล้ว SSH กลับเข้ามา (สำคัญ!)
exit
# SSH กลับเข้ามาใหม่
ssh user@your-server-ip
ต้อง logout แล้ว login ใหม่เพื่อให้สิทธิ์ Docker group มีผล ถ้าไม่ทำจะรัน docker ไม่ได้
7
ติดตั้ง Git
sudo apt install -y git
8
ตรวจสอบว่าทุกอย่างพร้อม
docker --version       # ต้องเห็น Docker version 24.x+
docker compose version # ต้องเห็น Docker Compose version v2.x+
git --version          # ต้องเห็น git version 2.x+
✓ ถ้าเห็นเลขเวอร์ชันทั้ง 3 คำสั่ง = เซิร์ฟเวอร์พร้อมแล้ว

ขั้นตอนที่ 2 — ตั้งค่า DNS

ไปที่เว็บ DNS provider (เช่น Cloudflare, Namecheap) แล้วเพิ่ม record ดังนี้:

TypeNameValuePriorityอธิบาย
Amail.example.comIP เซิร์ฟเวอร์ เช่น 45.76.1.2—บอกว่า mail.example.com อยู่ที่ IP ไหน
MXexample.commail.example.com10บอกว่าเมล์ @example.com ให้ส่งไปที่ mail.example.com
ตัวอย่างจริง: ถ้าโดเมนคือ tempmail.io แล้ว VPS IP คือ 45.76.1.2
→ A record: mail.tempmail.io → 45.76.1.2
→ MX record: tempmail.io → mail.tempmail.io priority 10

รอ 5-60 นาทีให้ DNS propagate แล้วตรวจ:

dig +short mail.example.com A     # ต้องเห็น IP เช่น 45.76.1.2
dig +short example.com MX         # ต้องเห็น: 10 mail.example.com.

ขั้นตอนที่ 7 — Nginx + SSL + Firewall

# ติดตั้ง
sudo apt install -y nginx certbot python3-certbot-nginx ufw

# สร้าง config file สำหรับ Nginx
sudo tee /etc/nginx/sites-available/tempmail-api <<'EOF'
server {
  listen 80;
  server_name api.example.com;
  location / {
    proxy_pass http://127.0.0.1:4000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    client_max_body_size 40M;
  }
}
EOF

# เปิดใช้งาน config
sudo ln -s /etc/nginx/sites-available/tempmail-api /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# ขอ SSL (ฟรี) — เปลี่ยน api.example.com เป็นโดเมนจริง
sudo certbot --nginx -d api.example.com

# ตั้ง Firewall
sudo ufw allow 22/tcp    # SSH (ต้องเปิดก่อน!)
sudo ufw allow 25/tcp    # SMTP รับเมล์
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable          # เปิด firewall
Nginx ทำหน้าที่เป็น reverse proxy: รับ HTTPS จากอินเทอร์เน็ต → ส่งต่อไป API ที่ port 4000
Certbot ขอ SSL certificate ฟรีจาก Let's Encrypt ต่ออายุอัตโนมัติ
UFW เป็น firewall เปิดเฉพาะ port ที่ต้องใช้ port อื่นปิดหมด
✓ เสร็จ! ระบบพร้อมใช้งานแล้ว เว็บหลักเรียก https://api.example.com/v1/* ได้เลย

เพิ่ม Node คืออะไร? ทำไมต้องเพิ่ม?

เมื่อระบบรับเมล์เยอะมาก หรือต้องการ failover (เครื่องหลักล่ม ยังรับเมล์ได้) ให้เพิ่ม Node ใหม่

ลองก่อน (MX 10) ถ้าเครื่อง 1 ล่ม (MX 20) อินเทอร์เน็ต ──▶ Node 1: mail-edge ──┐ อินเทอร์เน็ต ──▶ Node 2: mail-edge ──┤ ▼ Node 1: api + DB + Redis + worker
ℹ Node ใหม่รัน แค่ mail-edge ตัวเดียว ไม่มี Redis / Postgres ของตัวเอง
เชื่อมไปใช้ของเครื่องหลัก — เว็บหลักไม่ต้องเปลี่ยน config อะไรเลย

ขั้นตอนที่ 1 — เตรียมเครื่องหลัก

ต้องเปิดให้ Node ใหม่เชื่อมต่อ Redis, API, Rspamd ของเครื่องหลักได้

# บนเครื่องหลัก — เปิด firewall ให้ IP ของ Node ใหม่
# เปลี่ยน NODE2_IP เป็น IP จริงของ Node ใหม่
sudo ufw allow from NODE2_IP to any port 4000   # API
sudo ufw allow from NODE2_IP to any port 6379   # Redis
sudo ufw allow from NODE2_IP to any port 11333  # Rspamd
ตัวอย่าง: ถ้า Node ใหม่ IP คือ 103.28.70.50
sudo ufw allow from 103.28.70.50 to any port 4000
sudo ufw allow from 103.28.70.50 to any port 6379
sudo ufw allow from 103.28.70.50 to any port 11333

แก้ docker-compose.yml เปิด Redis port ให้ Node ใหม่:

# เพิ่มใน service redis:
  redis:
    ports:
      - "NODE2_IP:6379:6379"
docker compose up -d redis   # restart redis ด้วย port ใหม่

// เช็ค spam

  if (msg.isSpam) console.warn('⚠ Spam:', msg.subject);
